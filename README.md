# Glassifier Influencer Dashboard

Live dashboard for tracking 26.Q1&Q2 influencer marketing campaigns.

**Live URL:** https://yunseo-dev.github.io/glassifier-dashboard/

## How it works

```
Google Sheets (source of truth)
     │
     │  every 10 min
     ▼
GitHub Actions (.github/workflows/refresh-data.yml)
     │  runs scripts/build_data.py
     │  fetches all 7 published-CSV URLs
     │  parses + merges into data.json
     │  commits data.json back to repo
     ▼
GitHub Pages (this repo)
     │  serves index.html + dashboard.js + data.json
     ▼
Browser
     │  loads page → fetches data.json
     │  renders dashboard
     │  re-fetches every 10 min in the background
```

## Files

| File | Purpose |
|---|---|
| `index.html` | Page structure |
| `styles.css` | All styling |
| `dashboard.js` | Rendering logic — fetches `data.json` and updates the DOM |
| `chartjs.js` | Chart.js v4.5.1, bundled to avoid CDN dependency |
| `data.json` | Auto-generated by GitHub Actions; do NOT edit by hand |
| `scripts/build_data.py` | The script that builds `data.json` from Google Sheets |
| `.github/workflows/refresh-data.yml` | Schedule: runs every 10 min |

## Updating

### To change the dashboard appearance/logic

Edit `dashboard.js`, `styles.css`, or `index.html` and push. GitHub Pages picks up changes automatically.

### To add a new sheet/tab as a data source

1. In the Google Sheet, File → Share → Publish to web → select that tab → CSV → publish, copy the URL
2. Open `scripts/build_data.py`
3. Add the URL to the `SHEET_URLS` dict at the top
4. Write a parsing function for that tab's columns (use the existing parsers as a template)
5. Hook the parsed data into the final `out` dict at the bottom of `build()`
6. Push — the next scheduled run will include it

### To force a data refresh

Go to the Actions tab → "Refresh dashboard data" → "Run workflow" button.

### Local development

```bash
python3 scripts/build_data.py        # rebuild data.json locally
python3 -m http.server 8000          # serve at http://localhost:8000
```

## Notes

- The "today" reference for highlighting upcoming events is the actual current date (browser-side).
- Korean tab names are kept verbatim in the parsing code so the script keeps working when source data is in Korean.
- If `build_data.py` fails (e.g. Google's URL temporarily 403s), the workflow shows a red X but `data.json` is preserved — the dashboard keeps showing the last good data.
